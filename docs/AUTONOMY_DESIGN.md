# Ant Colony Wars — Autonomy Design

This document defines the autonomy model. Read it before writing any ant, job, or priority code.

---

## 1. Player Role

The player is the **colony brain**, not an ant controller.

| The player CAN | The player CANNOT |
|---|---|
| Place markers | Move an individual ant |
| Set colony priorities | Tell an ant to dig a specific tile |
| Approve room plans | Order an ant to attack a specific enemy |
| Choose upgrades | Directly change a tile |
| Issue defense or raid orders | Bypass the job queue |

Every player action communicates **colony-level intent**. Ants decide how to carry it out.

---

## 2. Ant Autonomy Rules

1. Every ant is always running its own FSM. No ant waits for player input to act.
2. Ants pick their next job by scoring all available jobs when they enter IDLE state.
3. The job queue is the only channel through which the world communicates with ants.
4. No script may call `ant.move_to()`, `ant.dig()`, or `ant.attack()` from outside the ant's own FSM.
5. If a job becomes unreachable mid-travel, the ant unclaims it and re-enters IDLE.
6. Ants ignore job types they cannot execute (soldiers do not score dig jobs).
7. An ant may be interrupted by an Emergency Marker. When the emergency is resolved, it returns to its previous job.
8. Ants must keep the colony alive by default. Low food, damaged rooms, exposed entrances, and missing nursery capacity create jobs through autonomy systems even when the player places no marker.
9. Autonomy systems may create jobs, change temporary emergency pressure, and update memory. They may not directly dig, gather, build, repair, spawn, or fight outside the job queue.

---

## 2.1 Colony Survival Autopilot

The colony should be playable as a self-running simulation. The player guides it with priorities and markers, but the ants handle survival details.

### Baseline Autopilot Goals

| Need | Autonomous response |
|---|---|
| Low food | Workers route-dig toward known or discovered food, then gather it |
| No useful jobs | Workers explore multiple tunnel frontiers instead of drifting one direction |
| Build priority high | Workers prepare space and execute Room Plan jobs |
| Repair priority above normal | Workers create REPAIR jobs for damaged structures |
| Defense priority high | Soldiers patrol queen, nursery, storage, and entrances |
| Queen damaged | Temporary emergency pressure redirects ants/soldiers until safe |

Rules:
- Autopilot creates intent only through jobs, priorities, and colony memory.
- Player-set priorities still matter; auto-escalation is temporary and must restore previous player settings.
- Food survival beats random exploration when stores are below the configured low-food ratio.
- Auto-created jobs must have budgets per category so the queue cannot flood or make workers fight between equally urgent jobs.
- Generated maps may create buried food and obstacles, but not free tunnels to food. Ants must dig those paths.
- The default start should not include a free surface shaft unless config enables it.

## 3. Priority System

Colony priorities control how strongly ants are attracted to each job category.
The player adjusts priorities from the Priority Panel in the HUD.

### Priority Categories

| Category | Controls |
|---|---|
| `food` | Food gathering and hauling jobs |
| `digging` | Dig Marker execution |
| `building` | Room Plan Marker execution (worker builds) |
| `nursery` | Egg care jobs generated by Nursery room |
| `soldiers` | Soldier training slots in Barracks |
| `defense` | Soldier patrol and threat-response behavior |
| `raid` | Soldiers pushing toward enemy territory |
| `repair` | Workers repairing damaged rooms/walls |

### Priority Levels

| Level | Weight Multiplier | Behavior |
|---|---|---|
| `low` | 0.5× | Ants work on this only when nothing higher is available |
| `normal` | 1.0× | Default |
| `high` | 1.5× | Ants prefer this over normal tasks |
| `emergency` | 2.5× | Ants drop current job and re-score immediately |

Priorities are stored as a `Dictionary` in `colony_state.gd`:
```gdscript
var priorities: Dictionary = {
    "food": "normal", "digging": "normal", "building": "normal",
    "nursery": "normal", "soldiers": "normal", "defense": "normal",
    "raid": "low", "repair": "normal"
}
```

When a priority changes to `emergency`, all ants currently on lower-priority jobs re-enter IDLE on their next `_process` tick.

## 4. Marker System

Markers are the player's only tool to shape ant behavior. All markers go through the job queue.

| Marker | How Placed | Who Claims | Effect |
|---|---|---|---|
| **Dig Marker** | Left-click dirt tile | Worker | Worker pathfinds and digs tile → becomes tunnel tile |
| **Gather Marker** | Left-click food source | Worker | Worker hauls food from source to nearest storage |
| **Room Plan Marker** | Click empty tunnel space, choose room type | Worker | Worker hauls materials and builds room over time |
| **Rally Marker** | Right-click location | Soldier | Soldiers move to location; engage enemies en route |
| **Raid Rally Marker** | Right-click enemy territory | Soldier | Soldiers push toward enemy queen; fight anything in the way |
| **Repair Marker** | Left-click damaged room/wall | Worker | Worker repairs structure; removes marker when done |
| **Emergency Marker** | Shift+right-click | Any idle ant | Overrides normal priorities; all idle ants score this job first |

Rules:
- Dig Markers cannot be placed on Queen Chamber tiles.
- Room Plan Markers can only be placed in cleared tunnel space.
- Rally Markers expire after a configurable timeout (soldiers return to patrol).
- Multiple Rally Markers stack — soldiers split between them proportionally.
- Only one Emergency Marker may be active at a time; placing a second removes the first.

## 5. Job Score

When an ant enters IDLE, it scores every unclaimed job and claims the highest score.

```
job_score = priority_weight(job.category)
		  + (10.0 / (distance_to_job + 1.0))
		  - (danger_level * 5.0)
		  + (resource_urgency * 3.0)
		  + (solo_bonus * 2.0)
```

| Term | Source | Notes |
|---|---|---|
| `priority_weight` | Colony priority level for the job's category | Low=0.5, Normal=1.0, High=1.5, Emergency=2.5 |
| `distance_to_job` | BFS tile steps from ant to job tile | 0 if already adjacent |
| `danger_level` | Number of known enemies within N tiles of the job | Workers avoid danger zones |
| `resource_urgency` | How critically the colony needs the job's output | 0–1 float, set by colony_state |
| `solo_bonus` | +1 if no other ant is already working this job category | Encourages variety |

Rules:
- A job with no reachable BFS path scores exactly 0 and is never claimed.
- Soldiers only score RALLY, RAID, PATROL, and FORTIFY jobs.
- Workers only score DIG, GATHER, BUILD, REPAIR, HAUL, and IDLE_WANDER jobs.
- If all jobs score 0, the ant enters an IDLE_WANDER micro-state (random short walk near home).

**Keep the formula simple.** Do not add terms without a clear gameplay reason.

## 6. Worker Behavior (FSM)

```
IDLE → MOVING → WORKING → IDLE
		 ↓ (no path found)
		IDLE
```

| State | What happens |
|---|---|
| `IDLE` | Score all unclaimed jobs. Claim best. Start moving. If no jobs: IDLE_WANDER for 2s, then re-score. |
| `MOVING` | BFS pathfind to job tile. Move one step per tick. If path blocked mid-travel: unclaim job, return to IDLE. |
| `WORKING` | Execute job action (dig pause, carry animation, build progress tick). When complete: remove/update job, return to IDLE. |
| `IDLE_WANDER` | Move to a random nearby tunnel tile. After arriving (or 2s): re-enter IDLE scoring. |

Workers do not fight. If a worker is within melee range of an enemy, it flees toward the queen.

---

## 7. Soldier Behavior (FSM)

```
IDLE → MOVING → FIGHTING → IDLE
  ↑                ↓ (enemy dead, no more threats)
  └────────────────┘
```

| State | What happens |
|---|---|
| `IDLE` | Check for enemies within detection radius. If enemy found: transition to FIGHTING. If Rally/Raid Marker exists: transition to MOVING toward it. Otherwise: patrol within N tiles of queen. |
| `MOVING` | Pathfind toward marker or patrol point. If enemy enters attack range during movement: transition to FIGHTING. |
| `FIGHTING` | Attack nearest enemy each tick. If enemy dies: scan for next threat. If no threats: return to IDLE. |

Soldier behavior scales with priorities:
- Defense=`emergency` → all idle soldiers rush toward any known enemy immediately.
- Raid=`high` or `emergency` → soldiers score Raid Rally jobs higher than patrol.
- Defense=`low` → soldiers only fight if directly attacked.

## 8. Room Plan Behavior

1. Player opens Room Plan menu (click in empty cleared tunnel).
2. Player selects room type and confirms.
3. A **Room Plan Marker** is placed at the location — visible as a blueprint overlay.
4. A BUILD job is added to the job queue for that room type.
5. Workers score the BUILD job like any other (affected by `building` priority).
6. The first worker to claim the BUILD job begins carrying resources to the site.
7. A build progress bar is visible on the Room Plan Marker.
8. When `progress >= build_cost` (from `data/rooms/{name}_config.json`): the room appears, marker removed.
9. If build is cancelled by the player, resources already delivered are lost.

The room does **not** appear instantly. `debug_mode: true` in `project.godot` may skip the timer for testing.

---

## 9. What NOT to Do

1. **Never call `ant.move_to()` from outside the ant's FSM.** Direction must come from the job queue, not from UI or world scripts.
2. **Never bypass the job queue.** Do not directly assign a target to an ant from any other script.
3. **Never place game logic in UI scripts.** HUD scripts read state; they never write it.
4. **Never make rooms appear instantly** (except in debug mode). All room construction flows through BUILD jobs.
5. **Never let the player remove tiles without a Dig Marker + worker.** There is no instant-dig shortcut.
6. **Never hardcode priority weights.** They must come from `colony_state.priorities`.
7. **Never add a new ant state** until the existing FSM states are fully stable.
8. **Never add a new marker type** mid-phase. Follow the phase plan.

## 10. Simple Examples

**Example A — Player wants more food**
1. Player sets `food` priority to `high` in Priority Panel.
2. `colony_state.priorities["food"]` changes to `"high"` (weight = 1.5).
3. All idle workers re-score jobs on next tick.
4. GATHER and HAUL jobs now score 1.5× higher than before.
5. Workers that were digging low-priority tunnels may switch to food gathering.
6. Player did not touch a single ant.

**Example B — Enemy approaching**
1. An enemy spider enters the map.
2. Nearest soldier's IDLE state detects enemy within detection radius.
3. Soldier transitions to FIGHTING, engages enemy.
4. If player manually sets `defense` to `emergency`, all idle soldiers rush toward the threat.
5. Player only changed a priority slider. Soldiers decided everything else.

**Example C — Building a Nursery**
1. Worker ants dig a 3×2 tunnel section.
2. Player opens room menu, places a Room Plan Marker of type "Nursery" in the cleared space.
3. BUILD job added to queue with `room_type = "nursery"`, `build_cost = 20 food`.
4. An idle worker scores the BUILD job, claims it, begins carrying food to the site.
5. After 20 food delivered: Nursery room appears, starts its egg hatch timer.
6. Player placed two things: Dig Markers and a Room Plan. Ants did everything else.
